Blinds Text Reveal - Property Controls

Text
The words or paragraph that will be revealed. Each line gets its own blind; line breaks in this field define the lines.

Trigger
When the animation runs: as soon as the component appears (Appear), or when it enters the viewport while scrolling (Layer in View). Use scroll for sections that should animate as the user scrolls.

Start
For scroll trigger: where the component must be in the viewport to start—Top (enters from top), Center (middle), or Bottom (enters from bottom). Lets you time the reveal with scroll position.

Replay
When using scroll trigger: if On, the animation runs again when the user scrolls back so the text is revealed each time. If Off, it only runs once.

Mode
Out: blinds move away to reveal the text. In–Out: blinds close over the text first, then open to reveal it (two-phase effect). Pick the style that fits your design.

Direction
Which way the blinds move: left-to-right, right-to-left, top-to-bottom, or bottom-to-top. Affects the feel of the reveal.

Alternate
If On, odd and even lines animate in opposite directions (e.g. odd left-to-right, even right-to-left). Gives a more dynamic, alternating reveal.

Order
Which line animates first: from first line to last, last to first, from center outward, or from edges toward center. Controls the sequence of the reveal.

Stagger
Time in seconds between each line starting. Higher values make the reveal feel slower and more deliberate; lower values make it snappier.

In
For In–Out mode only: how the “close” phase is animated (duration, easing). The blinds use this when they cover the text before revealing.

Out
How the “reveal” phase is animated (duration, easing). The blinds use this when they open to show the text. In Out-only mode this is the only phase.

Color
The color of the revealed text.

Blinds
The color of the blind strips that cover and reveal the text. Often matches or contrasts with the background.

Font
Typeface, size, weight, alignment, line height, and other text styling. What the revealed text looks like.

Tag
The HTML element used for the text (h1, h2, h3, h4, h5, h6, p, div, span). Use for semantics and accessibility—e.g. h1 for a main heading, p for a paragraph.
